# 기능 아이디어 백로그

브레인스토밍 단계에서 나온 아이디어 모음. 우선순위 없이 나열. 실제 구현 여부와
순서는 추후 결정.

## 확정 반영된 기능

- **자연어 질문 인터페이스**: "몇 년째 정체된 항목?" 같은 질문 → LLM이 파라미터
  추출 → SQL 집계 or 벡터검색 라우팅 → 답변 생성
- **정체(복붙) 탐지**: SQL 완전일치 + 벡터거리 기반 두 방식 병행
- **시민 피드백**: 이행보고 건별로 문제제기 + 근거자료 링크 남기기 (FEEDBACK 테이블)
- **용어사전**: 사전적 정의 + 프로젝트 맥락 설명, 팀원이 자유롭게 추가/수정 (TERM 테이블)
- **원문 보기**: source 필드(파일명+페이지) 기반으로 원본 PDF/이미지 페이지 바로 열기

## 검토/보류 중인 기능

### 검증 항목 자동 분해
"국정원장이 이행완료라는데 뭘 봐야 진짜인지 알 수 있을까?" 같은 질문에 대해,
시스템이 직접 사실 여부를 판단할 수는 없지만 "무엇을 확인해야 하는지" 체크리스트를
LLM이 제안하는 기능. 권고내용(+필요성)을 프롬프트에 넣고 "검증하려면 어떤 자료를
봐야 하는지 항목화해줘"로 구현 가능. 별도 도구 불필요, 프롬프트 설계만으로 가능.

### 권고 간 관계 (연관 권고 링크)
관련성 기준이 모호해 보류. 사람 수동 지정 / LLM 제안+승인 / 벡터 유사도 자동추천
중 방식 미정.

### 알림/구독 기능
특정 권고의 이행현황이 바뀌면(새 연도 보고서 파싱 시 전년도와 status가 달라지면)
팀원에게 알림. 이메일/슬랙 등 채널 미정.

### 감사로그(audit log)
IMPL, REC 등 원본 데이터를 팀원이 수정할 경우 "누가 언제 뭘 고쳤는지" 남기는 이력
테이블. TERM에는 이미 created_by/updated_by 반영, 다른 테이블 확장 여부는 미정.

### 자유 태그 시스템
권고사항에 자유 태그(예: "인권침해", "예산문제")를 다는 기능. TERM(용어사전)과는
별개로, 검색/필터링 편의를 위한 분류 목적.

### 첨부파일 관리
원본 PDF/이미지 파일 자체를 시스템에 보관하는 기능. "원문 보기"와 연계되나
파일 저장 위치(로컬 폴더 vs 파일서버)는 배포 단계에서 결정 필요.
